# Research — Background & Education

This folder holds the **background research and education** for the PolyWiz Exhibit Promo Tool concept. It is the "why and what exists" layer. The "what we'll build" layer (use cases, user tasks, integration design, checklist) lives in [`../concept/`](../concept/) and is authored with the user as the concept matures.

## Read in order

1. **[01-why-promote-open-calls.md](01-why-promote-open-calls.md)** — the education piece. What an open call is, why submission volume is the lever, the three gaps, the evidence that promotion moves the number, and why this is a systems problem worth a tool. **Start here.**
2. **[02-background-and-context.md](02-background-and-context.md)** — the people, the two June-2026 triggers, the "own the engine" reframe, how the ideas-inbox incubated this (Idea 011), and what's already decided.
3. **[03-related-systems.md](03-related-systems.md)** — the systems map: PolyWiz (engine), Artwork Archive (system-of-record), PressRanger (one feed), Zernio (rail), ideas-inbox (origin). Own vs. rent.
4. **[04-channel-directory.md](04-channel-directory.md)** — guide to the 66-channel verified directory: the five workstreams, the 15 columns, and how the tool uses it.
